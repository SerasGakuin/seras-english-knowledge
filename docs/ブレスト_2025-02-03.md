## 1. プロジェクトの背景

### 既存資産
- `seras-english-grammar` リポジトリに、参考書の書き起こしが進行中
- 核心・成川の2冊は書き起こし完了
- 他4冊（スクランブル、はじめの英文読解ドリル、肘井、入門英文問題精講）は目次のみ

### 塾の運営モデル
- 週1回・1時間の面談（生徒と大学生講師）
- 面談内容：1週間の勉強の仕方、質問対応、理解度チェック、来週の課題決定
- 基本は紙の参考書で自習させるスタイル

### 技術環境
- 生徒：紙ベース（参考書、ノート、確認テスト）
- 講師：Googleエコシステム（Docs、Sheets、GAS、Web app）
- 進捗管理はGoogle Sheetsで一元管理、APIで連携

## 2. 解決したい課題

### 最上位目標
**生徒の定着度を上げる**

### 「定着していない」の内訳

#### A. 理解の問題
- 読んだつもりで本質を掴めていない
- どこを理解すべきかわからない
- 「問題が解ければOK」という意識で、理論を理解しようという動機がない

#### B. 記憶の問題
- 理解したけど忘れる
- 復習のタイミングが体系化されていない

#### C. 応用の問題（文法→読解の接続不全）
- 文法知識と文構造意識（SVOCM、句・節）が繋がっていない
- 例：関係代名詞は知っているが、それが「形容詞節を作る」と知らない
- 例：分詞構文は知っているが、それが「分詞が副詞として働いている」と理解していない

### 構造的な問題

#### 参考書間の構成軸の違い
| 文法参考書（核心、成川、スクランブル） | 読解参考書（はじめの〜、肘井、入門精講） |
|---|---|
| 文法項目で章立て | 読解の視点で章立て |
| 仮定法、関係詞、不定詞... | SVの発見、カタマリ、識別、倒置... |

→ 生徒は自力で知識を接続できない

#### 講師の能力差
- 「理論が大事」という意識はある
- しかし限られた面談時間で「具体的にどこを、どう読め」まで落とし込めない
- 結果「問題が解けるだけじゃダメだよ〜」と一言で終わる

## 3. システムの方向性

### 基本思想
- 参考書学習が主、システムは補助
- 人の意識を変えるのは人、システムはそれを補助する
- 講師の能力に依存せず、枠組みでアシストする

### システムが担う役割

#### 「具体化」の代行
- 「この章を理解した」とは具体的に何ができる状態かを言語化
- 講師が説明しなくても、生徒が確認できる
- 講師も「これ説明できる？」と聞くだけで確認できる

#### 参考書の「標準化」
- 参考書に依存しない「あるべき知識体系」を定義
- 各参考書をそこにマッピング
- 抜け漏れや癖を補完する相互参照を提供

#### 文法と読解の「橋渡し」
- 文法用語と文構造用語の対応を明示
- 「関係代名詞 = 形容詞節を作る」を明確に伝える
- 文法→読解、読解→文法の双方向マッピング

## 4. 標準化された知識体系のイメージ

### 知識ノードの構造（例：関係代名詞）

```
■ 関係代名詞

【文法としての理解】
  → 核心 Part4_21 / 成川 Part07

【文構造での役割】
  → 形容詞節を作る
  → 修飾する名詞（先行詞）の直後に置かれる

【読解での応用】
  → 肘井 テーマ05「形容詞節で英文が読める」
  → はじめの〜 Chapter4-2「形容詞節① 関係代名詞」

【チェックポイント】
  □ 関係代名詞が作る節は何節か言える
  □ 先行詞を指して「これを修飾している」と説明できる
  □ who/which/thatの使い分けを説明できる
```

### 文法用語と文構造用語の対応表

```
【準動詞・関係詞 → 文中での役割】

関係代名詞 ──→ 形容詞節を作る（名詞を修飾）

分詞 ─┬─→ 形容詞として働く → 名詞を修飾
      └─→ 副詞として働く → 分詞構文

動名詞 ──→ 名詞として働く → S/O/Cになる

不定詞 ─┬─→ 名詞的用法 → S/O/Cになる
        ├─→ 形容詞的用法 → 名詞を修飾
        └─→ 副詞的用法 → 動詞/形容詞/文を修飾
```

## 8. 参考：書き起こしの構造

### 核心
- 1テーマ1ファイル（包括的）
- 例：`Part2_10_仮定法.md`（約25KB）

### 成川
- 1テーマ複数ファイル（細分化）
- 例：仮定法は8ファイルに分割

### 読解系（肘井、はじめの〜）
- 読解の視点で構成
- 「仮定法」という章はない
- 「倒置」「識別」などの章で文法知識が必要になる
