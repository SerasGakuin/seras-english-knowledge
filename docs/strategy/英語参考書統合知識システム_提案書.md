**英語参考書統合知識システム**

プロジェクト提案書

*〜 参考書学習の効果を最大化するための補助システム構築 〜*

# **1\. エグゼクティブサマリー**

## **1.1 プロジェクトの目的**

本プロジェクトは、Seras学院で使用している英語参考書（核心、成川英文法INPUT、スクランブル、はじめの英文読解ドリル、肘井の読解のための英文法、入門英文問題精講）の知識を体系的に統合し、生徒の学習効果を最大化するための補助システムを構築することを目的としています。

## **1.2 解決したい課題**

* 参考書間で知識が分断されており、生徒が自力で接続できない

* 「何がわかればOKか」というゴールが不明確なまま学習が進む

* 文法知識と読解スキルの接続ができていない生徒が多い

* 確認テストが「その場限り」で、体系的な復習ができていない

* 講師によって指導の粒度や確認ポイントにばらつきがある

## **1.3 目指す姿**

**「参考書学習が主、システムは補助」という原則のもと、以下を実現します：**

1. 生徒が「この範囲で何を理解すればOKか」を事前に把握できる

2. 参考書を横断して知識の抜け漏れを防ぐ

3. 最適なタイミングで復習を促し、定着率を最大化する

4. 講師の指導を標準化・効率化する

# **2\. 現状の課題（詳細分析）**

## **2.1 英文法教材の課題**

### **「入門英文法の核心」の問題点**

* 網羅度が低く、基礎的だが書いていないことも多い

* 講義本としては優秀だが、この1冊だけでは穴が残る

* 成川英文法などで補足が必要だが、どこを読むべきか体系化されていない

### **「成川英文法INPUT」の問題点**

* 分厚すぎて、自習室で開いたまま寝る生徒が続出

* 内容は良いが、全部読ませると挫折する

* 「必要な部分だけ参照」という使い方が確立していない

### **「スクランブル」の問題点**

* 1690問という量に圧倒されて、取り組みが中途半端になる

* 核心だけで理解が不十分な状態で演習しても効果が薄い

* どの問題が「今の自分に必要か」が見えにくい

## **2.2 英文解釈教材の課題**

### **「はじめの英文読解ドリル」の問題点**

* 例文は読めるが、そこで解説されている理論が習得できていない

* ドリル形式で「作業」として処理してしまう生徒が多い

* 核心やスクランブルで学んだ文法知識と接続されていない

### **教材間のレベルギャップ**

* 「はじめの英文読解ドリル」と「入門英文問題精講」の間に断崖がある

* 単語レベル、文章の長さ、複雑さが急に上がる

* 肘井を挟むことで緩和できるが、最適な順序が確立していない

# **3\. 提案するシステムの概要**

## **3.1 システムの全体像**

本システムは、Duolingoなどの語学学習アプリと同様の「間隔反復アルゴリズム」を基盤としつつ、「紙の参考書での自習」と「講師による対面指導」を組み合わせたハイブリッド型の学習支援システムです。

| コンポーネント | 内容 |
| :---- | :---- |
| 知識データベース | 6冊の参考書から抽出した知識ノードと、その関係性（前提・関連・包含）をネットワーク構造で管理 |
| ゴールチェックシート | 各参考書の章ごとに「何がわかればOKか」を明示したチェックリスト。生徒が自習時に横に置いて使用 |
| 確認テスト生成 | 生徒の進捗・習熟度に基づき、最適な問題を自動選択して印刷用テストを生成 |
| 習熟度管理 | 各生徒×各知識ノードの状態（学習日、正答率、復習タイミング）を記録・分析 |

## **3.2 生徒にとってのメリット**

5. 「何がわかればいいのか」が明確になり、漫然と読まなくなる

6. 自分で理解度を測れるので、自走力がつく

7. 参考書間の知識の繋がりを意識できる

8. 忘却曲線に基づいた復習で、定着率が向上する

## **3.3 講師にとってのメリット**

9. 「ちゃんとやってきたか」を効率的に確認できる

10. 生徒の弱点が可視化され、指導がピンポイントになる

11. 複数の参考書を横断した指導がしやすくなる

12. 確認テストの作成工数が大幅に削減される

# **4\. 具体的な使い方イメージ**

## **4.1 生徒の自習時**

生徒が参考書で学習する際、横にゴールチェックシートを置いて使用します。

**【ゴールチェックシート例】核心 第10章「仮定法」**

□ 仮定法過去と直説法過去の違いを説明できるか

□ 「If I were you」のwereが過去形である理由を説明できるか

□ 仮定法過去完了の形と意味を言えるか

□ wishを使った仮定法の文を作れるか

□ ifの省略と倒置のパターンを説明できるか

*→ 全部チェックできたら次へ。できないものがあれば核心p.XX、成川p.XXを復習*

## **4.2 面談時の確認テスト**

面談前に、システムが自動生成した確認テストを生徒に解いてもらいます。テストには以下が含まれます：

* 今週学習した範囲の本質理解を問う問題

* 関連知識との接続を確認する問題

* 2週間前など、復習タイミングが来た知識の定着確認

## **4.3 講師の口頭チェック**

テスト結果をもとに、講師が口頭で「なぜそうなるか」を確認します。知識データベースに基づいて、弱点に関連する知識も含めてフォローできます。

# **5\. 技術的な仕組み**

## **5.1 知識データベースの構造**

各参考書から抽出した「知識」を、ネットワーク構造で管理します。

### **知識ノード（Knowledge Node）**

最小単位の「知識」を表します。例えば「仮定法過去」「関係代名詞の制限用法」などがノードになります。

| 属性 | 説明 |
| :---- | :---- |
| 名前 | 仮定法過去、関係代名詞の制限用法、など |
| カテゴリ | 仮定法、関係詞、時制、など大分類 |
| 参考書の該当箇所 | 核心p.45-48、成川p.120-125、スクランブル\#312-320、など |
| 前提知識 | この知識を理解するために必要な他のノード |
| 関連知識 | 一緒に理解すると良い他のノード |

## **5.2 出題アルゴリズム**

Duolingoなどで使われている「間隔反復（Spaced Repetition）」アルゴリズムを採用します。SM-2アルゴリズムをベースに、以下の要素を考慮して出題を決定します：

* 今週学習した内容か

* 忘却曲線に基づく復習タイミングが来ているか

* 前回間違えたか

* 前提知識は定着しているか

* 関連知識との接続を確認すべきか

# **6\. 開発計画**

## **6.1 フェーズ分け**

### **Phase 1：知識データベース構築（1週間）**

まず「核心」と「成川英文法」の2冊から始め、知識ノードを抽出・体系化します。

13. 参考書のPDF化・OCR処理

14. テキストの構造化（章・節・項目の切り出し）

15. 知識ノードの粒度決定・抽出

16. ノード間の関係づけ（前提・関連・包含）

### **Phase 2：システム開発（2週間）**

17. データベース構築（Supabase/PostgreSQL）

18. 知識ネットワーク可視化機能

19. 確認テスト生成・印刷機能

20. 既存の生徒進捗データとの連携

### **Phase 3：運用・拡張（継続）**

21. 実際の運用を通じたアルゴリズムの調整

22. 残り4冊（スクランブル、はじめの〜、肘井、入門精講）の追加

23. 講師・生徒からのフィードバックに基づく改善

## **6.2 最初の一歩**

アジャイル開発の方針で、小さく始めて段階的に拡張します。まずは「核心」の1章分（例：仮定法）で試作を行い、粒度感や作業フローを確認します。

# **7\. 推奨する参考書ルート**

## **7.1 国公立・関関同立志望の場合**

### **高2秋まで（基礎完成フェーズ）**

**【文法】**

核心（メイン講義）＋ 成川（必要箇所のみ補足参照）＋ スクランブル（演習・定着）

**【読解】**

はじめの英文読解ドリル → 肘井・読解のための英文法必修編

### **高2冬〜高3春（応用準備フェーズ）**

入門英文問題精講

### **高3以降**

長文演習・過去問演習に集中できる状態を目指す

## **7.2 この設計のポイント**

* 核心だけでは穴がある → 成川を「辞書的に」補足使用

* スクランブルが重すぎる → 核心＋成川で理解した上で演習するので効率UP

* はじめの〜で理論が身につかない → 肘井を後に入れて「理論の言語化」を強化

* 入門精講とのギャップ → 肘井を挟むことで段階的に移行

# **8\. 期待される効果**

## **8.1 定量的な効果（期待）**

* 確認テスト作成工数：70%削減

* 生徒の文法定着率：20〜30%向上

* 高2終了時点での英語完成率：大幅に向上

## **8.2 定性的な効果**

* 生徒の自走力向上：「何を学ぶべきか」が明確になる

* 講師の指導品質標準化：誰が担当しても同じ確認ができる

* 塾の差別化：他塾にない体系的な学習支援を提供できる

# **9\. まとめ**

本プロジェクトは、「参考書学習が主、システムは補助」という原則のもと、生徒の英語学習の効果を最大化するための仕組みを構築するものです。

大学受験において英語は最も重要な科目であり、高2のうちにどれだけ仕上げられるかが合否を分けます。このシステムにより、生徒は「何を学ぶべきか」を明確に把握しながら自習を進め、講師は効率的に理解度を確認し、弱点をピンポイントでフォローできるようになります。

開発はアジャイル方式で、小さく始めて段階的に拡張します。まずは「核心」と「成川英文法」の2冊から知識データベースを構築し、実際の運用を通じて改善を重ねていきます。
