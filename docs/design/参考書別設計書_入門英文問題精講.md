## 2. 参考書の構造分析

### 2.1 全体構成

この参考書は**文法解説書ではなく英文読解の実践書**である。実際の入試問題を素材にして、英文の構造を読み解く力を養う。

```
基本講義（p.9-25）: 品詞・文型の基礎理論（13テーマ）
第1章　主語と動詞の把握（問1〜11: 11問）
第2章　準動詞（問12〜19: 8問）
第3章　関係詞（問20〜31: 12問）
第4章　接続詞（問32〜48: 17問）
第5章　比較（問49〜56: 8問）
第6章　その他の重要項目（問57〜72: 16問）
```

**冊構成**: 本冊（解答・解説）と別冊（問題文・語句）の2分冊

### 2.2 1セクション（=1問）の構造パターン

各「問」は以下の統一構造を持つ。他の参考書と根本的に異なり、**1〜3の英文 + その精密な構造分析・逐次解説**で構成される。

```
【別冊側】
  問題文: 1〜3の英文（番号①②③付き）
  出題大学: （津田塾大学）等
  語句注: *skyrocket「急増する」等
  読むヒント: 「主語はどこにある？」等

【本冊側】
  ┌─ 精講（構造分析）─────────────────────────┐
  │  英文に {S}{V}{O}{C} と ❶❷❸ マーカーを付与した │
  │  構造可視化バージョン                          │
  │  例: (❶With ...) {S}the ... {V}skyrocketed.   │
  └────────────────────────────────────────────┘
  ┌─ 精講（逐次解説）─────────────────────────┐
  │  ❶ 文の主語はどれ？ Point 〔副詞〕              │
  │    解説文（複数段落）                            │
  │    部分訳                                       │
  │  ❷ なぜコンマが必要？ 〔形容詞〕                │
  │    解説文                                       │
  │    部分訳                                       │
  │  ❸ 文の主語は garbage？ Point 〔前置詞〕         │
  │    解説文                                       │
  │    部分訳                                       │
  │  ...                                            │
  └────────────────────────────────────────────┘
  ┌─ 一言（コラム）────────────────────────────┐
  │  語源・文化背景・補足知識                        │
  └────────────────────────────────────────────┘
  ┌─ 解答例─────────────────────────────────┐
  │  全英文の完全な和訳                              │
  └────────────────────────────────────────────┘
```

### 2.3 特殊な構造要素

| 要素 | 説明 | 保存方針 |
|------|------|---------|
| **{S}{V}{O}{C}** | 主節の文の要素マーカー | sentence_structures の details に保存 |
| **(S)(V)(O)(C)** | 従属節・意味上の関係のマーカー | 同上 |
| **(　)** | 句の囲い | 同上 |
| **[　]** | 節の囲い | 同上 |
| **❶❷❸❹❺** | 説明ポイントのマーカー（精講と構造分析を対応） | Point として構造化（後述 §3.3） |
| **Point** | ❶❷❸の中で特に重要なもの | knowledge_nodes の priority を P1 にする根拠 |
| **〔品詞〕タグ** | 各 Point の分類（副詞/形容詞/前置詞/動詞/関係詞/接続詞 等） | knowledge_nodes の notes に記録 |
| **部分訳** | 各 Point での段階的な訳 | sentences の notes に記録 |
| **一言** | 語源・文化コラム | sections の notes に記録 |
| **読むヒント** | 別冊の学習ガイド | sections の notes に記録 |
| **出題大学** | 英文の出典 | sentences の notes に記録 |
| **語句ページ** | 発音・品詞・CERF・語根分析 | 後述 §6.1 |

### 3.3 ノード抽出ルール

この参考書は「問」単位で構成されており、各問の中に複数の「Point（❶❷❸）」がある。ノードは**Point の背後にある汎用的な知識概念**から抽出する。

#### 抽出の考え方

```
問1「主語の把握（前置詞＋名詞）」
  ❶ 文の主語はどれ？ Point 〔副詞〕   → 「文頭の前置詞句は副詞句」
  ❷ なぜコンマが必要？ 〔形容詞〕     → 形容詞の並列ルール（notes に記録、独立ノード化しない）
  ❸ 文の主語はgarbage？ Point 〔前置詞〕→ 「前置詞の付いた名詞は主語にならない」
  ❹ 文頭のWithの訳は？ 〔前置詞〕     → with の意味判別（notes に記録）

→ 抽出するノード:
  nsv-001「前置詞句による主語の識別」
    notes: 文頭の〈前置詞＋名詞〉は副詞句。前置詞の付いた名詞は主語にならない（a lot of 等の例外あり）。
           前置詞で始まる文は後ろのコンマ直後が主語。
```

#### 抽出ルール一覧

| 問の要素 | ノード化の方針 | 例 |
|---------|-------------|---|
| **Point マーク付きの❶❷❸** | **主要ノード**（P1）として抽出。同じ概念が複数問に登場する場合は1ノードに統合 | 「前置詞句による主語の識別」 |
| **Point なしの❶❷❸** | 関連する主要ノードの notes に記述。独立テスト可能なら別ノード（P2）に | コロンの働き、withの訳し方 |
| **複数問にまたがるテーマ** | 1つのノードで共通概念をカバーし、各問固有の側面は notes に追記 | 強調構文①②③ → 「強調構文」1ノード |
| **基本講義の各テーマ** | 基礎ノード（P1）。演習問題の前提知識 | 「品詞の働き: 名詞」「文型の判別」 |
| **一言（コラム）** | ノード化しない。sections の notes に記録 | 語源解説、文化背景 |

#### Point → ノードの統合基準

同じ概念が複数の問に登場する場合、以下の基準で統合・分割を判断する:

- **統合する**: 同一の文法ルールを異なる英文で適用しているだけの場合
  - 例: 問61/62/63「強調構文」→ 基本概念は同一。各問固有の応用パターンは notes に記録
- **分割する**: 同じ文法カテゴリだが、適用ルールが異なる場合
  - 例: 問1「前置詞句で主語を見誤る」vs 問9「文頭の長い副詞句」→ 別概念

### 3.4 粒度基準

| 指標 | 目安範囲 | 根拠 |
|------|---------|------|
| ノード/セクション比率 | 0.7〜1.5 | 1問が1つの明確なテーマを持つため、1問≒1ノード。複数問が同一ノードを共有する場合は比率が下がる |
| check_points/ノード | 3〜5 | 共通基準。Point の問い掛けから直接導出 |
| understanding_goals/ノード | 3〜6 | 共通基準 |

**想定ノード数**: 50〜65ノード（基本講義10 + 演習40〜55）

**分割判断基準**（共通）:
1. 独立テスト可能性: その Point だけで確認テストの問題を作れるか？
2. 部分習得可能性: 一方を知っていて他方を知らない状態がありえるか？
3. 十分な関連英文: そのノードに紐づく英文が3文以上あるか？

### 4.3 構文分析の方針

この参考書は精講で**著者自身の構造分析記法**を用いており、これをそのまま活用する。

```yaml
sentence_structures:
  - sentence_id: "ny-01-s1"
    label: "overall"
    value: "M S V"
    # 文型レベルの要約

  - sentence_id: "ny-01-s1"
    label: "details"
    value: "(With more and more disposable, packaged goods,)(M:副詞句) {S}the average household's volume of garbage(S) {V}skyrocketed(V)"
    # 精講の構造分析を変換。句()・節[]の表記を維持

  - sentence_id: "ny-01-s1"
    label: "partial_translations"
    value: |
      ❶ ますます多くの使い捨ての包装された商品
      ❸ 平均的な世帯のゴミの量は
      ❹ 使い捨ての包装された商品が増えると共に
    # 各 Point の部分訳を集約して記録
```

**label の種類**:
- `overall`: 文型記号（S V O C M）
- `details`: 精講の構造分析を変換したもの
- `partial_translations`: 各 Point の部分訳を ❶❷❸ と共に記録（この参考書独自）

### 4.4 Point 情報の保存

各問の Point（❶❷❸）は、この参考書の最も重要な教育コンテンツである。以下のように保存する:

| Point の要素 | 保存先 | 例 |
|-------------|--------|---|
| 問い掛け（「文の主語はどれ？」） | check_points の question | 「文頭に〈前置詞＋名詞〉がある場合、主語をどう識別するか？」 |
| 〔品詞〕タグ | knowledge_nodes の notes | 「Point分類: 〔副詞〕」 |
| 解説内容 | knowledge_nodes の notes | ノードの説明として構造化 |
| Point マーク（重要） | priority P1 の根拠 | — |
| 部分訳 | sentence_structures（partial_translations） | 上記 §4.3 参照 |

## 6. 情報保存の特記事項

### 6.1 語句ページの扱い

別冊の語句ページには、各英文に登場する単語の発音記号・品詞・CERF レベル・語根分析・関連語が詳細に記載されている。

**方針: 語句ページは構造化の対象外とする。**

理由:
- 本システムは文法・読解の知識構造を扱うものであり、語彙データベースは別の仕組みが適切
- 語句情報は英文ごとではなく単語ごとの粒度であり、本システムのデータモデルと合わない
- ただし、精講の解説中で語句の知識が文法理解に必須な場合は、knowledge_nodes の notes に記録する
  - 例: 「the poor = the + 形容詞 で『〜な人々』」→ これは語彙ではなく文法知識

### 6.2 この参考書特有の保存すべき情報

| 情報 | 保存先 | 保存方法 |
|------|--------|---------|
| **Point マーカー付き解説（❶❷❸）** | check_points + notes | question に問い掛け、answer に解説要旨。詳細は notes |
| **〔品詞〕タグ** | knowledge_nodes の notes | 「Point分類: 〔副詞〕〔前置詞〕」と記録 |

## 8. リスク・懸念事項

| リスク/懸念 | 対策案 |
|------------|--------|
| ノード統合の判断が難しい（例: 強調構文①②③を1ノードにするか3ノードにするか） | パイロットで実際に作成し、check_points の数で判断。1ノード3〜5個に収まるか？ |
| 精講の構造分析記法（{S}{V}等）の変換ルールが複雑 | 最初の5問で変換ルールを確立し、以降は統一 |
| 1問に複数英文がある場合、sentences と sections の関係が他の参考書と異なる | 1問 = 1セクション、1問内の複数英文 = 同一セクションの複数 sentences |
| 基本講義のノードと演習問題のノードの粒度が異なる可能性 | 基本講義は「前提知識ノード」（P1）、演習は「応用ノード」（P1/P2）と位置づける |
| 語句ページを対象外にすることで、語根分析などの有用情報が失われる | 精講の解説中で文法的に重要な語彙知識は notes に記録することで補完 |
| ページ番号が未確認（ここでは仮で2ページ/問として記載） | パイロット時にPDF画像で正確なページ範囲を確認し、セクション一覧を修正 |
