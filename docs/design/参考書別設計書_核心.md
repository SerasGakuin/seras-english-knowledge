## 2. 参考書の構造分析

### 2.1 全体構成

この参考書は**高校英文法の全体像を「核心」概念で貫く体系的な解説書**である。暗記ではなく理解重視の姿勢で、各テーマの最重要ポイント（核心）を1文で提示し、そこから体系的に展開する。

```
Part 1 > 品詞（テーマ01〜06: 冠詞・名詞・代名詞・形容詞・副詞・接続詞）
Part 2 > 動詞や時制（テーマ07〜11: 文型・現在形・完了形・仮定法・助動詞）
Part 3 > 準動詞（テーマ12〜16: 不定詞(1)(2)・動名詞・分詞・分詞構文）
Part 4 > その他の重要単元（テーマ17〜21: 受動態・比較・疑問詞・否定・関係詞）
```

### 2.2 1セクション（=1テーマ）の構造パターン

全21テーマが以下の**統一構造**を持つ。テーマの長さは7〜36ページと幅がある。

```
パターン（全テーマ共通）:

┌─ イントロダクション ──────────────────────┐
│  テーマの概要（キャッチーな見出し付き）        │
│  「中学英語との違い」（前提知識との差分を明示）  │
└────────────────────────────────────────┘

┌─ 核心ステートメント ──────────────────────┐
│  太字1文。テーマ全体を貫く最重要概念           │
│  ※ 1つの概念の場合と、2つの概念をスラッシュで  │
│  　 並べる場合がある                           │
│  例1: 核心 theは「共通認識」、aは「たくさんあるものの1つ」│
│  例2: 核心 分詞は「形容詞」の働き／-ingは能動、p.p.は受動│
│  例3: 核心 動名詞は「反復／中断／逃避」のイメージ       │
└────────────────────────────────────────┘

┌─ 本編解説 ───────────────────────────────┐
│  ≫ or ▷ 小見出し1                            │
│    解説文 + >例文 + ※補足 + 表・分類          │
│  ≫ or ▷ 小見出し2                            │
│    ...                                       │
│  ☑ 動詞ペア表 / 語彙チェックリスト             │
│    （lie vs. lay、感情動詞一覧 等）             │
│  ▶ まとめ（表形式の整理・比較）                │
│    ...                                       │
└────────────────────────────────────────┘

┌─ CHECK 入試問題に向けて ──────────────────┐
│  基礎確認問題（3〜4問、選択式/和訳）+ 解答・解説│
└────────────────────────────────────────┘

┌─ 実際の入試問題演習 ────────────────────┐
│  入試問題（3〜5問、出題大学名付き）+ 解答・解説  │
└────────────────────────────────────────┘
```

**見出しマーカーの使い分け**: テーマによって `≫` と `▷` が使われる。構造上の違いはなく、どちらもノード抽出の単位として同等に扱う。以降の説明で `≫/▷` と表記する。

### 2.3 特殊な構造要素

| 要素 | 説明 | 保存方針 |
|------|------|---------|
| **核心ステートメント** | テーマ冒頭の太字1文。テーマの最重要概念。1概念の場合と2概念をスラッシュで並べる場合がある | knowledge_nodes の notes に「核心: 〈内容〉」と記述。2概念の場合は最も関連するノードにそれぞれ記述 |
| **≫ 小見出し** | 本編解説の区切り。ノード抽出の自然な単位 | ≫ ごとにノード化を検討 |
| **▷ 小見出し** | ≫ と同じ役割の見出し（テーマ02,14等で使用）。構造上の違いなし | ≫ と同様に扱う |
| **☑ チェックリスト** | 動詞ペア表（lie/lay, rise/raise 等）、感情動詞一覧（3ページに及ぶ場合あり）、慣用表現リスト（look forward to -ing 等）など。単なるチェックではなく**独立テスト可能な知識を含むことが多い** | 内容の規模に応じて判断: (1) 数項目 → 関連ノードの check_points に変換、(2) 大規模リスト → P2ノードとして独立化（§3.3 参照） |
| **▶ まとめ** | 表形式の整理・比較 | notes に表の内容を記述 |
| **参考/補足** | 補足的な解説枠 | notes に記述 |
| **※ 注記** | 追加説明・注意点 | notes に記述 |
| **中学英語との違い** | 前提知識との差分を明示 | ノードの notes に「前提知識との差分: 〜」と記述 |
| **出題大学名** | 入試問題演習の出典 | sentences の notes に「出典: 〇〇大学」と記録 |
| **> 例文** | 本文中の英文（和訳付き） | sentences に登録（role: example） |
| **表・分類** | 語彙リスト・分類表・比較表 | notes に要点を記述。表全体の再現は不要 |

### 3.3 ノード抽出ルール

この参考書は「テーマ」単位で構成されており、各テーマ内に複数の「≫ 小見出し」がある。ノードは**≫ 小見出しの背後にある汎用的な知識概念**から抽出する。

#### 抽出の考え方（テーマ別の具体例）

```
テーマ01「冠詞」（7p・シンプル）
  核心: theは「共通認識」、aは「たくさんあるものの1つ」
  ≫ theの基本的な考え方        → khs-001「定冠詞theの核心（共通認識）」
  ≫ theを使った会話表現        → khs-001 の notes に記録（独立テスト困難）
  ≫「正体不明」なものには"a"    → khs-002「不定冠詞aの核心（たくさんあるものの1つ）」
  ≫ "a"の色々な意味            → khs-002 の notes に記録（aの派生用法）
  → 2ノード

テーマ08「現在形」（11p・中規模）
  核心:「現在形」は「現在・過去・未来形」、「進行形」は「途中」
  ≫ 現在形は「現在・過去・未来形」  → kvt-XXX「現在形の核心（時制を超えた不変の事実）」
  ≫ 現在形の5つの用法              → kvt-XXX の notes に用法一覧（具体用法は核心原理から派生）
  ≫ 進行形は「途中」               → kvt-YYY「進行形の核心（途中）」
  ≫ 進行形の5秒ルール              → kvt-YYY の check_points に（判別基準として独立テスト可能）
  → 2〜3ノード（核心ステートメントの2概念がそのまま2ノードに対応）

テーマ14「動名詞」（16p・中規模・▷使用）
  核心: 動名詞は「反復／中断／逃避」のイメージ
  ▷ 動名詞は「反復／中断／逃避」    → kjd-XXX「動名詞の核心（3つのイメージ）」
  ▷ 不定詞（to do）との使い分け     → kjd-XXX の notes に対比説明
  ▷ to doと-ingで意味が変わる動詞   → kjd-YYY「to do/-ing で意味が変わる動詞」（独立テスト可能な判別知識）
  ▷ 慣用表現（look forward to -ing等）→ kjd-ZZZ「動名詞の慣用表現」（P2ノード、リストが大規模）
  → 3ノード（▷マーカーだが≫と同等に扱う）

テーマ15「分詞」（11p・☑大規模リスト含む）
  核心: 分詞は「形容詞」の働き／-ingは能動（～する）、p.p.は受動（～される）
  ≫ 分詞は「形容詞」→ 名詞修飾・補語  → kjd-XXX「分詞の核心（形容詞としての働き）」
  ≫ -ingは能動、p.p.は受動             → kjd-YYY「-ing/p.p.の使い分け（能動/受動）」
  ☑ 感情動詞チェック（3p、30語以上）    → kjd-ZZZ「感情動詞の-ing/p.p.使い分け」（P2ノード ※大規模リストのため独立化）
  → 3ノード（☑からも独立ノード抽出）

テーマ07a「文型」（前半・分割テーマ）
  核心: 品詞がわかれば、文型がわかり、意味がわかる
  ≫ 3つのフェイズ（品詞→句節→文型）     → kvt-XXX「文の分析3フェイズ」
  ≫ 自動詞と他動詞の判別               → kvt-YYY「自動詞/他動詞の判別（「何を？」テスト）」
  ☑ lie/lay, rise/raise 等の動詞ペア   → kvt-YYY の check_points + notes に（判別ノードに紐付け）
  ≫ 第1文型・第2文型                   → kvt-ZZZ「第1・2文型の特徴」
  → 3ノード（CHECK・入試問題はKaku_07bに属する）
```

#### 抽出ルール一覧

| テーマ内の要素 | ノード化の方針 | 例 |
|-------------|-------------|---|
| **核心ステートメント** | ノード化しない。最も関連するノードの notes 冒頭に「核心: 〈内容〉」と記述。2概念並記の場合は各ノードにそれぞれ記述 | 「核心: theは「共通認識」」 |
| **≫/▷ 小見出し（主要概念）** | **主要ノード**（P1）として抽出。独立テスト可能な概念のみ | 「定冠詞theの核心」 |
| **≫/▷ 小見出し（派生・応用）** | 関連する主要ノードの notes に記述。独立テスト可能なら別ノード（P2） | 「theを使った会話表現」→ notes |
| **☑ 小規模（数項目）** | 関連ノードの check_points に変換 | 「lie vs. lay の活用と意味の違い」→ check_points |
| **☑ 大規模（リスト10項目超/3ページ超）** | **P2ノードとして独立化**。テスト可能な判別知識として扱う | 「感情動詞の-ing/p.p.使い分け」→ 独立ノード |
| **▶ まとめ表** | 関連ノードの notes に内容を記述 | 全体否定 vs. 部分否定の比較表 |
| **参考/補足** | 関連ノードの notes に記述 | 「自動詞・他動詞（形式的な説明）」 |
| **中学英語との違い** | 関連ノードの notes に「前提知識との差分: 〜」と記述 | 「中学では省略されるが高校では〜」 |
| **慣用表現リスト** | リストが大規模（10項目超）なら P2 ノードとして独立化。少数なら notes | 「動名詞の慣用表現」→ P2ノード |
| **動詞のペア表** | 関連ノードの check_points + notes | 自動詞/他動詞の判別ノードに紐付け |

#### ノード統合の基準

同一テーマ内の複数の ≫ 小見出しが同一概念を扱う場合:

- **統合する**: 同一の文法原理を異なる側面から説明しているだけの場合
  - 例: ≫ theの基本的な考え方 + ≫ theを使った会話表現 → 1ノード（「theの共通認識」が核心、会話表現は応用）
- **分割する**: 異なる文法ルールを扱い、独立テスト可能な場合
  - 例: ≫ 全体否定の2パターン vs. ≫ 部分否定 → 2ノード（判別基準が異なる）

### 3.4 粒度基準

| 指標 | 目安範囲 | 根拠 |
|------|---------|------|
| ノード/セクション比率 | 2.0〜4.0 | 各テーマに複数の ≫ 小見出しがあり、主要概念が2〜4個含まれる |
| check_points/ノード | 3〜5 | 共通基準 |
| understanding_goals/ノード | 3〜6 | 共通基準 |

**想定ノード数**: 60〜90ノード（21テーマ × 平均3〜4ノード）

**分割判断基準**（共通）:
1. 独立テスト可能性: その ≫ 小見出しの内容だけで確認テストの問題を作れるか？
2. 部分習得可能性: 一方を知っていて他方を知らない状態がありえるか？
3. 十分な関連英文: そのノードに紐づく英文が3文以上あるか？

## 4. 英文設計

### 4.1 英文抽出ルール

| 英文の種類 | 登録する/しない | role | 備考 |
|-----------|--------------|------|------|
| `>` 付き例文（和訳あり） | **登録する** | example | 本文中の解説に使われる完全な英文 |
| CHECK問題の英文 | **登録する** | check | 基礎確認。空所を埋めた完全形で登録 |
| 入試問題演習の英文 | **登録する** | exam | 出題大学名を notes に記録 |
| 表中の例語句（once a week 等） | 登録しない | — | 句・フレーズであり完全な文でない |
| 解説中の断片（語句のみ、活用表） | 登録しない | — | 文として不完全 |
| 日本語のみの例 | 登録しない | — | 英文がない |
| `>`マーカーなしだがノードの唯一の実例 | **登録する**（例外） | example | decisions.yamlに判断根拠を記録。完全な文で和訳が明確な場合に限る |

**想定英文数**: 300〜400文（例文 ~200 + CHECK ~70 + 入試問題 ~80）

### 4.3 構文分析の方針

この参考書には精密な構文分析記法はない。ただし品詞・文型の解説が詳しいため、文型情報を活用する。

```yaml
sentence_structures:
  - sentence_id: "kaku-01-e01"
    label: "overall"
    value: "S V M"
    # 文型レベルの要約

  - sentence_id: "kaku-01-e01"
    label: "details"
    value: "<The sun>(S) rises(V) [in the east](M:副詞句)"
    # 句のカタマリを<>()[]で示し、役割を注記
```

**label の種類**:
- `overall`: 文型記号（S V O C M）
- `details`: 句・節のカタマリと役割の分析

### 5.4 セクション間の前提関係（section_prerequisites）

この参考書は**テーマごとに独立性が高い**設計（文法項目別の解説書）。ただし以下の依存関係がある:

| 前提セクション | 依存セクション | 理由 |
|--------------|-------------|------|

## 8. リスク・懸念事項

| リスク/懸念 | 対策案 |
|------------|--------|
| テーマの長さの幅が大きい（7p〜36p）。分割テーマの境界が適切でない可能性 | パイロットで分割テーマ（Kaku_07a）を含めて検証。必要に応じて分割位置を調整 |
| 例文が番号なしで本文中に散在し、抽出判断が曖昧 | `>` マーカー付きの完全な英文のみ登録。断片・語句は登録しない |
| 核心ステートメントのノード化判断 | ノード化しない（独立テスト不可）。最関連ノードの notes 冒頭に記述 |
| 表・分類リストの情報が膨大（特に助動詞・比較） | 表全体を再現するのではなく、テスト可能な知識ポイントを check_points に抽出 |
| 「中学英語との違い」の情報保存 | 生徒が混同しやすいポイントとして notes に記述。テスト不可能な導入的内容は省略可 |
| CHECK と入試問題演習の英文は空所補充形式。完全形に復元する必要 | 解答を参照して空所を埋めた完全な英文として登録 |
| 著者の比喩表現（セレブ/庶民、トランプのジョーカー等）の扱い | 比喩自体は保存不要。比喩が説明している文法ルールを正確に notes に記述 |
| テーマ07「文型」は品詞解説も含み、他テーマとの重複リスク | 品詞の基本はテーマ01〜06のノードに集約し、テーマ07では文型視点でのみノード化 |
| ☑ の語彙リストが非常に大規模な場合がある（テーマ15 感情動詞チェック: 3ページ30語超） | 大規模リストは P2 ノードとして独立化。リスト全項目の notes 記載ではなく、代表例 + 判別基準の check_points 化で対応 |
| ▷ マーカーと ≫ マーカーの混在（テーマによって異なる）| 構造上の違いなし。両者を同等に扱い、設計書では ≫/▷ と表記 |

### 9.3 発見事項と設計書への影響

#### (A) 分割テーマ前半の英文数
- Kaku_07aは例文のみ9文。CHECK・入試問題がKaku_07bに集中する。
- **影響**: 想定英文数がやや下振れする可能性。Phase 2で分割テーマ後半（Kaku_07b等）の英文数を確認し、合計で十分か検証。
- **設計書変更**: 不要。リスク§8に「分割テーマの前半にCHECK・入試問題がない場合がある」を追加済み（decisions #4）。

#### (B) 入試問題の登録判断
- 対話形式の出題（Do you have the time?）や断片（Twice a month）は登録対象外。
- **影響**: 入試問題数が想定より少なくなる可能性。ただし品質（完全文のみ登録）が優先。
- **設計書変更**: §4.1の英文抽出ルールは既に「完全な文でないものは登録しない」と明記。追加不要。

#### (C) check_points上限5件の制約
- kvt-002は5件のcheck_pointsで動詞ペアの主要項目をカバー。grow up/bring up, sit/seatはnotesでカバー。
- **影響**: テーマ07のような情報量が多いノードでは check_points 上限に達する。
- **設計書変更**: 不要。優先度の高い項目をcheck_pointsに、その他をnotesに記録する運用で問題なし。

#### (D) 「>マーカーなし」例文の扱い
- Kaku_07aの「The patient waited quietly until his name was called.」は>マーカーなしだが登録した（decisions #3）。
- **設計書変更**: §4.1に補足追記が望ましい。「例外: マーカーなしでも、ノードの唯一の実例として機能する完全な英文は登録可（decisions.yamlに記録）」

### 9.4 Phase 2に向けた推奨事項

1. **バッチサイズ**: 3セクション/バッチが適切。品質ゲート通過後にチェックポイント書き出し。
2. **Part 1 → Part 4 → Part 2 → Part 3 の順**: パイロットで確認したPart順に進行。
3. **分割テーマ**: 前半/後半をできれば同バッチで処理し、合算での英文数を確認。
4. **validate.py**: Gate 2 の自動バリデーションに使用。ID命名規則のチェックは参考書ごとに調整が必要。
