## 2. 参考書の構造分析

### 2.1 全体構成

この参考書は**文法解説書ではなく、文法・語法の整理書＋問題集**である。入試で頻出の文法事項を261のThemeに分類し、各Themeに2〜6の4択問題を配置。解説と Power Up! で知識を体系化する。

**本書の根本的な性格**: 知識は問題の解説と Power Up! 表に埋め込まれており、独立した「概念解説」セクションは存在しない。他の解説書（核心・成川）とは対照的に、**問題→解説→まとめ表**の構造で知識を伝える。

### 2.2 1セクション（= 1 Theme）の構造パターン

#### パターンA: 文法・語法 Theme（Part 1-2, 大多数）

```
Theme XX 〈テーマ名〉
  ├── 問題 NNN 〈難易度〉〈出題大学〉
  │     問題文（4択/穴埋め/整序/正誤判定）
  │     選択肢 ①②③④
  │     解答
  │     日本語訳
  ├── 問題 NNN+1 ...
  ├── ...（2〜6問）
  ├── 解説
  │     各問題の解説（太字の正解形 + 解説文 + 比較・語句・選択肢の補足）
  ├── Power Up! XXX（任意: 0〜3個）
  │     まとめ表/チェックリスト/ルール一覧
  └── プラスα（任意）
        補足例文（完全な英文 + 和訳）
```

#### パターンB: 語い・イディオム Theme（Part 3-4）

パターンAと同構造だが、解説がより語彙/フレーズ中心。各問題が個別の単語/イディオムを問う形式で、Power Up! に大規模な語彙リストがつくことが多い。

#### パターンC: 会話表現 Theme（Part 5）

```
Theme XXX 〈場面名〉
  ├── 問題 NNNN 〈出題大学〉
  │     A: 日本語文 + 英文（穴埋め）
  │     B: 日本語文 + 英文（穴埋め）
  │     解答
  ├── ...
  ├── 解説
  │     各問題の会話パターン解説（太字の定型表現 + 比較/重要マーク）
  └── Power Up! XXX
        応答パターン一覧（☐チェックリスト形式）
```

#### パターンD: 特別編（Special edition）

```
特別編 N 〈テーマ名〉
  ├── 問題 (N) 〈指示文〉
  │     英文（下線部あり）
  │     選択肢 ①②③④（各選択肢が完全な英文で下線部を含む）
  ├── ...（3〜4問/セクション）
  ├── 解答（全文の和訳付き）
  ├── 解説（各選択肢の文法用法を詳細に分析）
  └── CHECK!（文法識別のまとめルール）
```

### 2.3 特殊な構造要素

| 要素 | 説明 | 保存方針 |
|------|------|---------|
| **〈基本〉〈頻出〉〈発展〉〈超頻〉** | 問題の難易度/頻出度マーカー | sentences の notes に記録 |
| **Power Up! NNN** | まとめ表・チェックリスト（☐付き）・ルール一覧 | knowledge_nodes の notes/check_points に統合 |
| **プラスα** | 補足例文（完全な英文＋和訳） | sentences として登録（role: example） |
| **〈比較〉** | 類似表現との対比 | knowledge_nodes の notes に記録 |
| **〈語句〉** | 語彙注釈 | sentences の notes に記録 |
| **〈選択肢〉** | 誤答選択肢の解説 | knowledge_nodes の notes に記録 |
| **〈重要〉** | 重要ポイントの強調 | check_points の優先的な項目とする |
| **(⇨ Theme XX)** | Theme間の相互参照 | knowledge_nodes の notes に記録 |
| **(⇨ Power Up! XX)** | Power Up!への参照 | 同上 |
| **(⇨ 問題番号)** | 問題番号への参照 | 同上 |
| **CHECK!** | 特別編のまとめルール | knowledge_nodes の notes に記録 |
| **出題大学名** | 問題の出典 | sentences の notes に記録 |
| **問題番号（連番）** | 書籍全体で連番（1〜約1400） | sentences の number フィールド |

### 2.4 対象外とする部分

**Part 6 発音・アクセント（Ch30-31）は構造化の対象外とする。**

理由:
- 発音・アクセントは音韻学的な知識であり、本システムの文法・読解の知識モデルと根本的に異なる
- 問題形式が「下線部の発音が他と異なるものを選べ」であり、タグ付け可能な英文が存在しない
- 知識ノードの check_points / understanding_goals が「発音記号を知っているか」になり、確認テスト生成に不適合
- 共通テストでの発音問題出題頻度も低下傾向

### 3.3 ノード抽出ルール

この参考書は「問題集」であり、知識は解説と Power Up! に分散している。ノードは**Theme の文法概念**と **Power Up! のまとめ知識**から抽出する。

#### 抽出の考え方

```
Ch01 時制
  Theme 1「基本3時制の用法」(問題 1-5)
    解説: 現在形=習慣・真理、過去形=過去の事実、未来 will/be going to
    Power Up! 1: 「3時制の基本用法まとめ」
    → sdv-001「基本3時制の用法」
      check_points: 現在形=習慣/真理、過去形=過去の事実、will/be going to の違い

  Theme 2「進行形の用法」(問題 6-9)
    解説: 現在進行形=一時的動作、過去進行形=過去の進行中の動作
    Power Up! 2: 「進行形にできない動詞一覧」
    → sdv-002「進行形の用法と制約」
      check_points: 進行形の基本用法、進行形にできない動詞（状態動詞）

  Theme 3「進行形にできない動詞」(問題 10-13)
    解説: 状態動詞（know, believe, belong...）は進行形不可
    → Theme 2 の sdv-002 に統合（同一テーマ）
```

#### 抽出ルール一覧

| Theme の要素 | ノード化の方針 | 例 |
|-------------|-------------|---|
| **Theme タイトルの文法概念** | **主要ノード**（P1/P2）として抽出。テーマ名が知識の核心 | 「基本3時制の用法」→ P1ノード |
| **同一概念の連続 Theme** | 1ノードに統合。notes に各 Theme の固有側面を記録 | Theme 2+3（進行形の用法+制約）→ 1ノード |
| **Power Up! のまとめ表** | 最も近い Theme ノードの notes/check_points に統合 | PU! 1「3時制の基本用法」→ sdv-001 の notes |
| **Power Up!（独立的な知識）** | 関連 Theme と概念が異なる場合は別ノード（P2） | PU! 2「状態動詞一覧」→ 大規模なら別ノード |
| **解説中の〈重要〉マーク** | check_points に優先登録 | 「would rather は仮定法」→ check_point |
| **解説中の〈比較〉** | 関連ノードの notes に記録 | 「Nice to meet you. vs Nice meeting you.」 |
| **プラスα の例文** | sentences に登録、関連ノードにタグ付け | — |
| **相互参照（⇨ Theme XX）** | prerequisites リンクの候補 | — |

#### ノード統合の基準

- **統合する**: 連続 Theme が同一文法概念の異なる側面を扱う場合
  - 例: Theme 21+22（助動詞 + have done ①②）→ 1ノード「助動詞の完了形」
  - 例: Theme 2+3（進行形の用法 + 進行形にできない動詞）→ 1ノード「進行形」
- **分割する**: 同一章内でも文法概念が独立している場合
  - 例: Theme 1（基本3時制）と Theme 4（現在完了）→ 別ノード
- **統合比率の目安**: 連続 Theme の統合は最大3 Theme まで。4 Theme 以上の統合は粒度が粗すぎる

### 3.4 粒度基準

| 指標 | 目安範囲 | 根拠 |
|------|---------|------|
| ノード/セクション比率 | **0.5〜1.0** | 問題集のため Theme 間の重複が多い。同一概念の連続 Theme が頻繁に統合される |
| check_points/ノード | 3〜5 | 共通基準。Power Up! のチェック項目から直接導出 |
| understanding_goals/ノード | 3〜6 | 共通基準 |

**想定ノード数**: 120〜200ノード（271セクション × 平均0.5〜0.7）

内訳:
- Part 1 文法（124 themes）→ 60〜85ノード
- Part 2 語法（59 themes）→ 25〜40ノード
- Part 3 語い（26 themes）→ 10〜18ノード
- Part 4 イディオム（41 themes）→ 15〜25ノード
- Part 5 会話表現（11 themes）→ 5〜10ノード
- Special edition（10 sections）→ 5〜8ノード

**粒度基準が他参考書と異なる理由**: 核心(2.0〜4.0)や成川(1.0〜2.0)は解説書のため1セクション内に複数の独立概念がある。本書は問題集であり、各 Theme が1つの焦点を持つため、Theme 間で統合が生じる方向に粒度が下がる。入門英文(0.7〜1.5)に近い性格。

**分割判断基準**（共通）:
1. 独立テスト可能性: そのTheme の文法概念だけで確認テストの問題を作れるか？
2. 部分習得可能性: 一方を知っていて他方を知らない状態がありえるか？
3. 十分な関連英文: そのノードに紐づく英文が3文以上あるか？

### 4.2 難易度・頻出度マーカーの定義

この参考書には**明示的な難易度マーカーが存在する**（他参考書と異なる最大の特徴）。

| マーカー | 参考書での表記 | 保存先 | 保存値 |
|---------|-------------|--------|--------|
| 難易度（基本） | 〈基本〉 | sentences.notes | `難易度: 基本` |
| 頻出度（頻出） | 〈頻出〉 | sentences.notes | `難易度: 頻出` |
| 頻出度（超頻出） | 〈超頻〉 | sentences.notes | `難易度: 超頻出` |
| 難易度（発展） | 〈発展〉 | sentences.notes | `難易度: 発展` |

**priority（ノードレベル）への反映**:
- P1: Theme の核心概念。〈頻出〉〈超頻〉の問題が集中する Theme
- P2: 応用・慣用表現。〈発展〉寄りの Theme
- P3: 補足・例外

### 4.3 構文分析の方針

この参考書には精密な構文分析記法はない。問題の解説で文法ポイントを説明するが、文型記号の体系的な付与はない。

```yaml
sentence_structures:
  - sentence_id: "scr-001-p01"
    label: "overall"
    value: "S V M"

  - sentence_id: "scr-001-p01"
    label: "details"
    value: "<Water>(S) boils(V) [at 100 degrees Celsius](M:前置詞句)"
```

**label の種類**:
- `overall`: 文型記号（S V O C M）
- `details`: 句・節のカタマリと役割の分析

### 振り返り・設計の妥当性

1. **粒度基準（0.5-1.0）は妥当**: 各 Theme が1つの焦点を持つため、1 Theme = 1ノードの対応が自然に成立した。連続 Theme の統合はパイロット範囲では発生しなかったが、本格構造化で検証が必要
2. **問題集特有の構造化パターンが確立**:
   - 穴埋め問題 → 解説の太字正解部分から完全な英文を復元 → exam として登録
   - Power Up! の語句リスト（表形式） → notes にテキスト化して記録（sentences 非登録）
   - Power Up! の例文（完全な文＋和訳） → sentences に example として登録
   - 問題文が複数文の場合 → 全文を english フィールドに含める（Gate 4 で指摘・修正済み）
3. **設計書の修正は不要**: ノード抽出ルール・英文抽出ルール・ID体系が3セクションとも問題なく機能した
4. **Gate 4 での修正2件**: (a) 複数文問題の第2文欠落、(b) PU!93 の相互参照 Theme 497 の欠落。いずれも注意点として本格構造化に引き継ぐ

## 8. リスク・懸念事項

| リスク/懸念 | 対策案 |
|------------|--------|
| **規模が巨大**（271セクション・120-200ノード・700-1000英文） | フェーズ分割: Phase 1 = Part 1-2 + Special（193セクション）、Phase 2 = Part 3-5（78セクション） |
| **ノード統合判断が連続 Theme 間で頻発** | 統合基準を厳格に適用: 最大3 Theme 統合、check_points 3-5個に収まるか確認 |
| **問題文が穴埋め形式で完全な英文でない** | 解説の太字正解部分から完全な英文を復元して登録。復元不可能な場合は登録しない |
| **Power Up! が約210個あり、ノードへの割り当てが複雑** | 原則: 最も近い Theme のノードに統合。独立テスト可能な大規模 PU! のみ別ノード化 |
| **Part 3-4（語い・イディオム）の知識モデルが文法と異なる** | Phase 2 で扱う。ノードの粒度を「テーマグループ」（例: 「意外な意味の自動詞」）に設定 |
| **会話表現（Part 5）の英文が対話形式** | 完全な文のみ sentences に登録。短い応答（"Sure." 等）はノードの notes に記録 |
| **Power Up! 番号が概算** | パイロットで Ch01(PU!1-2), Ch07(PU!29), Ch15(PU!92-93) を確認済み。概算と一致。本格構造化で各章ごとに確認を継続 |
| **ページ番号が未確認** | パイロットでは未確認。本格構造化でPDF画像から確認してセクション一覧に追加する |
