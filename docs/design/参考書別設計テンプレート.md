# 参考書別設計テンプレート

> 各参考書の構造化に着手する**前に**、このテンプレートに沿って設計書を作成する。
> 設計書は人間のレビューを経て承認を受けてから、構造化作業に着手する。
> ファイル名: `参考書別設計書_〈参考書名〉.md`

---

## 設計書を作成する手順

1. 書き起こしMDを**全体通読**する（最低でも各章の冒頭 + 代表セクション3つ以上）
2. PDF画像も確認し、図解・表・マーカーの使われ方を把握する
3. 以下のテンプレートの各セクションを埋める
4. 人間にレビューを依頼し、承認を受ける
5. パイロット（最初の2〜3セクション）後に、必要に応じて改訂する

## 2. 参考書の構造分析

### 2.1 全体構成

> 参考書がどのような章・パート構成になっているかを記述する。

```
例:
Part 1: 品詞（テーマ01〜05）
Part 2: 文型（テーマ06〜10）
...
```

### 2.2 1セクションの構造パターン

> 参考書の各セクション（テーマ/レッスン/ユニット）がどのような構造を持っているかを分析する。
> 複数のパターンがある場合はすべて列挙する。

```
例（はじめの英文読解ドリルの場合）:
  パターンA「始める前に」: 概念説明 → 用語定義 → ルール列挙
  パターンB「ドリル」: ポイント解説 → ドリル1(例文) → ドリル3(練習) → ドリル4(発展)

例（核心の場合）:
  パターン: イントロダクション → 核心ステートメント → 本編解説 → 例文(番号付き) → 問題
```

### 2.3 特殊な構造要素

> その参考書に特有の構造的特徴を列挙する。

| 要素 | 説明 | 例 |
|------|------|---|
| | | |

```
例:
- 「核心」ステートメント: 各テーマの冒頭に1文で核心を要約
- 樹形図: 文型判別の分岐を視覚化
- ☑マーク: 重要ポイントの強調
- ※注記: 補足説明
- 「〜するところの」: 著者が否定する教え方への注意
```

### 3.2 ID体系

> ノード・セクション・英文のID命名規則を定義する。

| 対象 | 形式 | 例 |
|------|------|---|
| **knowledge_nodes** | `{カテゴリ接頭辞}-{3桁連番}` | `khn-001` |
| **sections** | `{参考書接頭辞}_{章番号}_{セクション番号}` | `Kaku_01_01` |
| **sentences** | `{参考書接頭辞}-{セクション番号}-{role短縮}-{ドリル番号}-{連番}` | `kaku-01-e1-01` |

#### role短縮の定義

| role | 短縮 | 説明 |
|------|------|------|
| | | |

```
例:
| example     | e | 例文（著者が解説付きで示す英文） |
| practice    | p | 練習問題 |
| advanced    | a | 発展問題 |
| explanation | x | 解説中の補足的な英文 |
```

### 3.3 ノード抽出ルール

> この参考書の構造パターンに応じて、どこからどのようにノードを抽出するかを定義する。

| セクション内の要素 | ノード化の方針 | 例 |
|------------------|-------------|---|
| | | |

```
例（核心の場合）:
| 「核心」ステートメント     | 概念定義ノード（P1）       | 「何節か？」「完全か不完全か？」 |
| 本編の各小見出し（≫付き）  | 具体的知識ノード（P1/P2）   | 関係代名詞は「形容詞の働き」 |
| 表・分類（グループ分け）   | リストノード（P2）          | 関係詞の3グループ分類 |
| ※注記・補足              | 親ノードの notes に記述     | 「〜するところの」は応急処置 |
| 発展的内容               | P3ノードまたは notes        | 連鎖関係詞 |
```

### 3.4 粒度基準

> この参考書でのノード粒度の目安を定義する。

| 指標 | 目安範囲 | 根拠 |
|------|---------|------|
| ノード/セクション比率 | | |
| check_points/ノード | 3〜5 | 共通基準 |
| understanding_goals/ノード | 3〜6 | 共通基準 |

```
例:
| ノード/セクション比率 | 2.0〜3.5 | 核心は各テーマが深く、小見出しが多い |
```

**分割判断基準**（共通）:
1. 独立テスト可能性: そのポイントだけで確認テストの問題を作れるか？
2. 部分習得可能性: 一方を知っていて他方を知らない状態がありえるか？
3. 十分な練習英文: 分割後も各ノードに3文以上の英文があるか？

## 5. セクション設計
